## 2026-05-22 — Wave 4 kickoff (post-Wave-3 verification)

Wave 3 finished with all 8 stories self-reporting `passes: true`, but per the
established discipline this got an auditor's run before being declared truly
done. The audit found:

1. `test_real_providers_full_game.py` (US-071) had never actually been run.
   First execution exposed a counting bug in the parse-rate quality gate:
   `parse_rate = ok_count / total` excluded `fallback_ok`, treating a healthy
   primary→fallback path (Cerebras failed, DeepInfra fallback returned valid
   JSON) as a parse failure. Real run: 40 ok + 31 fallback_ok = 71/71
   parseable, gate reported 56% and failed.
2. The same test's "every seat must produce a non-NOOP / non-ABSTAIN action"
   assertion was unachievable in legitimate games: a villager who ABSTAINs
   on Day 1 and dies Night 1 has no further chances (Day Discussion forces
   NOOP, villagers cannot inspect/protect/kill).

Fixed pre-Wave-4 as commit `884076c`:
- New `_PARSED_OK_STATUSES = {"ok", "fallback_ok"}` constant used across
  all three real-provider integration tests (day-1 sanity, full game,
  mini-gauntlet).
- Action-coverage assertion loosened from "every seat" to "at least
  len(seats) - 2 seats" (5 of 7), preserving the silent-coercion detection
  intent while tolerating early-game eliminations.

Real-run evidence post-fix:
- US-071: winner=MAFIA reason=PARITY_REACHED calls=70 parse=100% cost=$0.66
- US-073: games=3 winners=[MAFIA,MAFIA,MAFIA] calls=210 parse=100% cost=$3.08

Total real-money cost across the verification: $3.74.

The three-MAFIA-wins sample in US-073 is too small to draw conclusions

## 2026-05-22 — US-079 (same-model multi-host fallback)

Landed by maintainer (not Ralph). The architectural slice that makes the
rest of wave-4 possible.

What shipped:
- `RoutingPolicy.same_model_hosts: tuple[SameModelHost, ...]` — alternate
  provider endpoints serving the SAME upstream weights. Each carries
  `provider`, `litellm_model_id`, `auth_secret_ref`, and optional
  `api_base`.
- `LiteLlmAdapter` now iterates primary → same_model_hosts → fallback.
  Per-host retry budgets are isolated (a primary exhausting its 3
  attempts does NOT consume any same-model host's budget). Per-host
  `api_base` and `api_key` are forwarded to `litellm.acompletion` only
  when set, so the primary call keeps its pre-US-079 kwargs bit-for-bit.

  `_PARSED_OK_STATUSES` in all three real-provider integration tests.

### Script bug: COMPLETE-sentinel false positive

The patched prompt's `<promise>COMPLETE</promise>` rule fired correctly
— Codex ran `jq '[.userStories[] | select(.passes==false)] | length'
ralph/prd.json`, saw `4`, and DID NOT emit the sentinel. Despite that,
the script printed "Ralph completed all tasks!" and exited.

Root cause: `grep -q "<promise>COMPLETE</promise>"` in
`ralph/ralph-codex.sh` (and `ralph/ralph.sh`) matches the LITERAL
string anywhere in stdout, including inside the prompt text itself
where the sentinel is mentioned as part of the stop-condition
instruction. The prompt echoes through stdout on every Codex
invocation, so the false positive is structural.

Fix: tighten the pattern to require the sentinel on its OWN LINE:
`grep -qE '^[[:space:]]*<promise>COMPLETE</promise>[[:space:]]*$'`.
Patched in both scripts. The next Ralph fire should now pick US-081
(partial work already in working tree from this run + the previous
run) and continue without exiting after one iteration.

## 2026-05-28 — US-083 (Heterogeneous real-LLM gauntlet: one distinct model per seat)

Turned Padrino from a self-play stress test into a head-to-head benchmark:
each of the 7 seats now runs a DISTINCT model identity.

Design (minimal-invasion):
- `src/padrino/llm/multiplex.py` — `SeatMultiplexAdapter` implements the
  `LlmAdapter` Protocol and dispatches each `complete()` to the adapter
  assigned to `observation.you.player_id`. The runner's `_RecordingAdapter`
  only ever calls `complete()`, so NO `run_game` signature change was
  needed — heterogeneity is fully encapsulated in the multiplex.
- `src/padrino/gauntlets/heterogeneous.py` — `build_heterogeneous_adapter`
  takes a seat -> `AgentBuild` mapping and builds one `LiteLlmAdapter` per
  seat with a SINGLE-host `RoutingPolicy` (no fallback, no same-model host),
  resolving auth/base_url from `provider_endpoints(settings)`.
- `SeatMultiplexAdapter.calls_by_seat` accumulator enables the per-model
  coverage assertion (`AdapterResult` carries no seat id).

Tests:
- `tests/llm/test_multiplex.py` (5) + `tests/gauntlets/test_heterogeneous.py`
  (4) — non-integration, monkeypatched fake keys (resolve_secret reads env
  only, no network).
- `tests/integration/test_heterogeneous_gauntlet.py` — real single game,
  $4 cap, parse-rate >=70%, per-model coverage, evaluate_gauntlet
  rating_deltas per model.

- **Stale-`.coverage` trap:** plain `uv run pytest` does NOT write coverage

Padrino's first real scheduler job: cron-driven recurring tournaments.

- Migration `0014_scheduled_gauntlets.py` + `ScheduledGauntlet` model
  (name unique, schedule_cron, roster_spec_json, n_games, cost_cap_usd,
  enabled, last_run_at, last_run_gauntlet_id, next_run_at).
- `src/padrino/db/repositories/scheduled_gauntlets.py` — CRUD + list_due
  (enabled AND next_run_at NULL/past, Python-side tz-aware filtering).
- `src/padrino/core/scheduling/` — HAND-ROLLED pure-core 5-field cron:
  `next_run_at(cron_expr, *, after)` (deterministic, no wallclock; after
  injected) + `humanize_cron`. croniter is NOT a dep; the parser supports
  */n, a-b, a-b/n, lists, and the DOM-or-DOW rule. 10 unit tests
  (every-Nth-minute/hour, month boundary, leap day, UTC-DST stability,
  OR semantics, invalid specs, humanize).
- `src/padrino/scheduler/{gauntlet_job,bootstrap}.py` — the job fires due
  rows via run_tournament_from_roster, finalizes (or stamps 'cost_capped' on
  overrun), recomputes next_run_at. `run_scheduler` gained a `tick_hook` seam;
  bootstrap wires the job in.
- Admin POST/PATCH/DELETE + scrubbed public GET (`schedule_cron_human`, no
  raw cron, no cost cap). DELETE = soft-disable (row kept for audit).
- Docs: docs/deployment/scheduled-gauntlets.md.

Firewall note: `core/scheduling` needs `datetime` for pure cron arithmetic
(after is injected, never read from the clock). Per `test_purity.py`'s own
"update this test deliberately" guidance, added a NARROW allowlist
(scheduling -> {datetime}) AND a new `test_allowed_datetime_modules_make_no_
wallclock_calls` that AST-bans `.now()/.utcnow()/.today()` — net-strengthening
the firewall rather than loosening it.

recomputed.

### Learnings for future iterations

- `finalize_gauntlet_if_done` commits internally — do NOT wrap it in
  `session.begin()` (double-commit). Cost-capped path sets status manually
  inside its own begin() block.
- The cron helper's UTC-only contract is deliberate: the scheduler clock is
  UTC, so civil-DST cron semantics are out of scope (documented). A future
  per-timezone schedule would need UTC<->local normalization.
