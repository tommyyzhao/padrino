## Codebase Patterns

- `create_app(public_surface_only=True)` (or `PADRINO_PUBLIC_SURFACE_ONLY=true`) builds the internet-facing process: only the public router + health router are mounted; private routers and `/metrics` are NOT registered, so private prefixes 404 (route absent), never 401/403. The flag resolves like the other `create_app` knobs: explicit kwarg wins, else `settings.padrino_public_surface_only`. Note `scheduled_gauntlets_router` carries BOTH private CRUD and one public route (`/public/scheduled-gauntlets`); gating the whole router (per AC) drops that public route too — the must-work public surface is the enumerated set (`/public/live`, `/public/recent`, `/public/ladder`, `/public/games/{id}/live`).
- `src/padrino/public/` is the new module for Wave 7 public surface. Start each new public module here.
- The public spectator website (`web/dashboard`) must call ONLY `/public/*` endpoints from any nav- or URL-reachable page (it ships against the public-surface-only API). To get a league_id without the private `/gauntlets`, chain public reads: `/public/recent` -> `GET /public/games/{id}.league_id`. The e2e smoke spawn sets `PADRINO_PUBLIC_LEADERBOARD_ANONYMOUS=true` (`tests/e2e/global-setup.ts`) so keyless `/public/*` reads work; the smoke harness does NOT seed broadcastable RECENT games, so `/public/recent` + `/public/live` are empty there (only `/public/ladder` has data from the gauntlet's ratings).
- `padrino.core.spectator_projection.project_event_for_spectator` is the identity-blind filter for PUBLIC events; always build on top of it rather than reimplementing visibility filtering.
- `padrino.core.observation_privacy.FORBIDDEN_PAYLOAD_KEYS` is the canonical deny list for identity/role/faction leaks — shared across spectator, ranked-observation, and public_event_v1.
- Golden-file tests live in `tests/public/fixtures/` as JSON. The test loads the file and diffs computed output against stored expected — any schema drift fails the test until the golden file is updated consciously.
- `gauntlets.claim_oldest_pending` is the single scheduler claim seam (called from `runner/scheduler.py`). It applies `with_for_update(skip_locked=True)` ONLY when `session.get_bind().dialect.name == "postgresql"`, making horizontal scaling safe on Postgres (two replicas never double-claim) while leaving SQLite single-writer. For any new dialect-conditional SQL in a repo, branch on `session.get_bind().dialect.name`. To test cross-replica DB concurrency, give each claimer its own `factory()` session + `commit()` and `asyncio.gather` them (a shared session serializes and proves nothing); reuse the testcontainers `postgres_url`/`postgres_engine` fixtures from `tests/db/test_dialect_portability.py` under `@pytest.mark.postgres`.
- `padrino.observability.alerts.AlertNotifier` is the canonical operational-alert seam: `await notifier.fire(key, **details)` is fire-once-per-transition (de-duped via a per-key active flag) and `notifier.resolve(key)` re-arms it; it always logs `alert.fired` and POSTs JSON to `padrino_alert_webhook_url` when set. Build it with `build_alert_notifier(settings)`. The matchmaking tick (`run_continuous_matchmaking_tick`, optional `notifier=` param, threaded from `scheduler/bootstrap.py` + `cli.py`) is where spend-cap / stale-heartbeat / guard-unavailable / admission-denial-streak conditions are wired. ## Codebase Patterns

Verdict order for ONE held message: (1) pure `public.moderation.deterministic_first_pass_message(text)` instant backstop — a hard-tier hit (`_TOXIC_PATTERNS`, sentinel `toxic_word`) is an immediate BLOCK and the async guard is NEVER called; (2) pure `core.agents.sanitizer.sanitize_visible_text` + pure `public.moderation.deterministic_span_mask(text)` (masks `_SOFT_MASK_PATTERNS`, sentinel `mask_word`, with same-length `*` runs) produce the SOFT_MASK release text; (3) async `MessageGuardAdapter.check_message` under `asyncio.wait_for(timeout_s)` — `False`->BLOCK, timeout/error->FALL BACK to the step-2 deterministic verdict (the game never halts), safe->ALLOW/SOFT_MASK. HARD-tier and soft-tier are SEPARATE pattern sets so SOFT_MASK is reachable (a hard hit blocks before span-mask runs). A BLOCK carries `cleaned_text=None` and is never released/chained/sidecar'd; ALLOW/SOFT_MASK route `cleaned_text` to the US-123 sidecar (raw never inline in a hashed payload). Test the gate with a RECORDING guard stub (asserts `guard.seen == [...]`) to prove it actually saw the message (guards the Wave-7 vacuous-gate bug class).

Pattern: pure producer in core (core/turing/scoring.py — no clock/random/IO; emits accuracy as a DECIMAL STRING, never a float, so it crosses the core boundary cleanly), thin impure shell (api/human_turing.py) that resolves the caller's seat via GameSeat.occupant_principal_id (->403 wrong_seat), requires terminality from the REPLAYED state (state.terminal_result is not None, ->409), builds the seat truth from GameSeat.seat_kind==HUMAN (AI_TAKEOVER counts as AI, matching the reveal), scores, and persists once-per-guesser (UNIQUE(game_id, guesser_public_id); a re-submission returns the stored row — never re-score). GATING a personal result behind an action: a GET endpoint returns the stored stat or 404 if the prerequisite (the submitted guess) is absent — the reveal hands out the accuracy ONLY after the viewer submits their own guess. NO leaderboard in v1 (decision 6): the scorer only ever takes ONE guesser + the truth, never a cross-player ranking.
