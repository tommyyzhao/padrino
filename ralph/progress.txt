## Codebase Patterns
(Reusable learnings discovered across iterations. Keep this section short and general — story-specific notes live in the iteration entries below.)

- Project layout: `src/` layout under `src/padrino/`. Tests live in `tests/`. Use `from padrino.X import Y`.
- Quality gates: `uv run ruff check . && uv run ruff format --check . && uv run mypy src tests && uv run pytest -m "not integration"`. CI runs the same.
- Pure core: nothing under `src/padrino/core/` may import from `padrino.db`, `padrino.llm`, `padrino.api`, `padrino.runner`, or call wall-clock / network / random. Inject impure dependencies via constructor.
- Hash chain: `event_hash = sha256(prev_event_hash + canonical_json(event_without_hash_and_timestamp))`. Timestamps and the hash field itself are excluded from the hash input.
- Mafia chat is never parsed for mechanics — `action` field only. Tests must enforce this.
- Ranked observations never include model identity, ratings, gauntlet clone index, or other games' transcripts.
- Use `from __future__ import annotations` at the top of every module. Pydantic v2 models with `model_config = ConfigDict(frozen=True)` for state/event types.

- pydantic-settings: use `Settings(_env_file=None)` in tests so monkeypatch controls env vars without the on-disk .env interfering.
- structlog configure_logging uses a module-level `_configured` flag for idempotency.
- Mypy strict rejects module-level `def f(x: T) -> U` as a satisfier of a Protocol method (positional-only/staticmethod mismatch). Expose ruleset behaviour as `Final[dict[K, V]]` attributes (e.g. `ROLE_FACTIONS`) and have engine code read the mapping directly. Optional `*_for(...)` wrappers around the dict are fine for ergonomics, but the Protocol contract should be over the dict.

## 2026-05-14T00:45 - US-004
- Implemented `src/padrino/core/engine/rng.py`: `SeededRng(seed: str | bytes)` with stateful counter-based expansion via `sha256(seed_bytes + counter.to_bytes(8, "big"))`. Methods: `next_bytes`, `randbelow` (mask-and-reject for unbiased uniform), `shuffle` (Fisher-Yates returning a new list), `choice`.
- `tests/core/test_rng.py`: 23 tests covering determinism across instances, range bounds, statefulness, deterministic shuffle/choice, immutable inputs, and an AST assertion that `rng.py` does not import `random` or `secrets`.
- Files changed: `src/padrino/core/engine/rng.py`, `tests/core/test_rng.py`
- **Learnings for future iterations:**
  - Ruff `RUF023` requires `__slots__` tuple entries to be sorted (natural order). Easy to miss when slots are listed in declaration order.
  - For unbiased `randbelow(n)`, use `bit_length = (n - 1).bit_length()`, draw `ceil(bit_length / 8)` bytes, mask high bits to `(1 << bit_length) - 1`, then reject values `>= n`. Rejection probability is bounded at 50% per draw.
  - `SeededRng` is stateful: each `next_bytes`/`randbelow`/`shuffle`/`choice` advances the internal counter. Downstream callers should construct a fresh `SeededRng` per logical purpose (e.g., `SeededRng(sha256(b"roles" + game_seed))` for role assignment) to avoid coupling unrelated draws.
