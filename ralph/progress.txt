## Codebase Patterns
(Reusable learnings discovered across wave-2 iterations. Keep this section short
and general â€” story-specific notes live in the iteration entries below. Patterns
inherited from wave 1 are NOT duplicated here; read the archive for those.)

- Terminal-game detection (US-049): a game is "terminal" iff `Game.status == 'COMPLETED'`. The runner sets this atomically with the `GameTerminated` event row inside `GamePersistence._persist_terminated_event`. Do NOT add new `event_type='GameTerminated'` queries â€” filter on `Game.status` instead. The legacy event-scan pattern is gone from `padrino.api` and `padrino.gauntlets` and should not be reintroduced.
- `Game.terminal_result` is a JSON column shaped `{winner, reason, day_terminated}`. The separate `Game.terminal_reason` column was dropped â€” reason lives inside the JSON. Read it via `game.terminal_result["winner"]`.
- Game-status string values are UPPERCASE to match the existing Gauntlet convention: `CREATED` â†’ `RUNNING` â†’ `COMPLETED`. Story copy showed lowercase as an illustrative example; the codebase uses uppercase.
- Atomic terminal-write seam: `_persist_terminated_event(persistence, stored, state, ranked, day_terminated)` writes the event row, `Game.status='COMPLETED'`, `Game.terminal_result`, and (when applicable) every rating update inside ONE `session.begin()`. Tests assert rollback by monkeypatching `update_ratings_for_game` to raise â€” the game row and event row must NOT be persisted.
- Seat-backfill seam: `_persist_roles_assigned(persistence, stored)` writes the `RolesAssigned` event row and every `game_seats` row in ONE txn. Requires `GamePersistence.agent_builds` to cover EVERY seat in the payload; partial mappings silently skip seat insertion (the event row still goes through) so callers driving the runner without a per-seat build mapping don't crash. Demo gauntlets and full integration tests always supply the complete map.
- The runner forbidden-imports firewall test (`test_game_runner_does_not_import_forbidden_modules` in `tests/runner/test_game_runner.py`) blocks `random`, `secrets`, `datetime`, `time` imports from `src/padrino/runner/game_runner.py`. If a story needs wall-clock at the runner level, factor the helper into a sibling module (e.g. `padrino.runner.persistence` or similar) and import the helper â€” don't import the stdlib module from `game_runner.py`.
- Secret resolution (US-050): `padrino.llm.secrets.resolve_secret(ref)` is the single seam that turns `ModelProvider.auth_secret_ref` strings into credentials. Supported schemes are `env:VAR_NAME` and `file:/abs/path` ONLY (no `vault:`, no relative paths, no `~` expansion). Raise `SecretResolutionError` on every failure mode â€” never return `None` or `""`. The adapter resolves ONCE at `__init__` and stashes the value on `self._auth_secret`; never re-resolve per request. The admin POST /model-providers route also resolves at creation so a bad ref 422s immediately instead of 401-ing at game time.
- The `padrino.llm.litellm_adapter` AST firewall test (`test_litellm_adapter_does_not_import_db_or_random`) checks `mod.startswith(forbidden)` for forbidden module names including `"secrets"`. The stdlib module is `secrets`; our project module is `padrino.llm.secrets`. Because the prefix check is anchored at the start of the dotted name, `padrino.llm.secrets` does NOT match the `secrets` prefix â€” safe to import. Future stdlib `secrets` (token generation, etc.) imports from the adapter would still be caught.
- Recorded-cassette tests (US-051): use `vcrpy` (committed dev dep, ignored by mypy via the `vcr.*` override). The `live_llm` marker is registered in `pyproject.toml` and default-skipped by the `pytest_collection_modifyitems` hook in `tests/conftest.py`. Opt-in is either `--live-llm` (CLI flag) or `-m live_llm` (marker selector). Cassette helper: `vcr.VCR(record_mode="once" if PADRINO_RECORD_LLM=="1" else "none", match_on=("method","scheme","host","path"), filter_headers=[...], before_record_request=_scrub_request, before_record_response=_scrub_response)`. Loose `match_on` lets cassettes ignore differing request bodies across model parameters. The audit test `test_cassettes_have_no_secret_shaped_substrings` scans every committed yaml for `sk-...`, `sk-ant-...`, and `Bearer <token>` patterns and is unmarked so it runs on every default invocation.
- Probing litellm's transport: vcrpy 8.x intercepts the async httpx transport via `vcr.stubs.httpcore_stubs` (the older `httpx_stubs` module was renamed). Litellm dispatches to provider-specific URLs which can be observed by monkeypatching `httpx.AsyncClient.send`. Confirmed URLs as of the wave-2 commit: openaiâ†’`api.openai.com/v1/chat/completions`, anthropicâ†’`api.anthropic.com/v1/messages`, cerebrasâ†’`api.cerebras.ai/v1/chat/completions`, deepinfraâ†’`api.deepinfra.com/v1/openai/chat/completions`, ollama (with `ollama/<model>` prefix, NOT `ollama_chat/`)â†’`localhost:11434/api/generate`. The ollama provider's `/api/generate` returns `{"response": "<text>", ...}` (not the `/api/chat` shape with `message.content`).

## 2026-05-14T23:55 - US-050
- New module `padrino.llm.secrets` with `resolve_secret(ref) -> str` and `SecretResolutionError`. Supports `env:VAR_NAME` (errors on missing, empty, or whitespace-only) and `file:/abs/path` (rejects tilde and relative paths; rejects missing files, non-regular files, and any file with `g/o` r/w bits set; strips trailing whitespace; errors on empty result).
- `LiteLlmAdapter.__init__` now requires a keyword-only `auth_secret_ref: str`; it calls `resolve_secret(...)` exactly once and stashes the result on `self._auth_secret` (added to `__slots__`). Per-request `complete()` never re-reads the secret. POST /model-providers in `padrino.api.routes.admin` also runs `resolve_secret` at creation so a misconfigured deployment 422s loudly rather than 401-ing later.
- Demo gauntlet passes `auth_secret_ref="env:CEREBRAS_API_KEY"`; integration test `tests/integration/test_real_providers.py` matches. `tests/api/test_admin_routes.py` gets an autouse fixture that monkeypatches CEREBRAS_API_KEY + X env vars so the resolve-at-creation pass succeeds. New tests `tests/llm/test_secrets.py` (16 cases) and two new admin tests cover the unresolvable-ref and unknown-scheme 422 paths.
- Files changed: `src/padrino/llm/secrets.py` (new), `src/padrino/llm/litellm_adapter.py`, `src/padrino/api/routes/admin.py`, `src/padrino/demo_gauntlet.py`, `tests/llm/test_secrets.py` (new), `tests/llm/test_litellm_adapter.py`, `tests/api/test_admin_routes.py`, `tests/integration/test_real_providers.py`, `ralph/prd.json`, `ralph/progress.txt`.
- **Learnings for future iterations:**
  - `monkeypatch.setattr(os.environ, "__getitem__", ...)` does NOT work â€” `os._Environ` doesn't accept attribute reassignment on bound methods. To count secret resolutions in a test, monkeypatch the seam itself: `monkeypatch.setattr(padrino.llm.litellm_adapter, "resolve_secret", counting_resolve)`. That's also why tests verify "single resolution" by patching the symbol the adapter imported, not by intercepting env reads.
  - FastAPI dep order: when a handler has both `body: Model` and `session: AsyncSession = Depends(get_session)`, the dep + body parse complete BEFORE handler body runs. So `test_admin_routes_rejected_when_no_session_factory` still 503s (dep fails first) even though the handler would 422 on an unresolvable secret â€” order of evaluation saves us. Don't rely on this for tests that need to assert the resolve_secret path; assert against bodies that pass dep+pydantic gates first.
  - File-permission policy is stricter than the story's literal text: we reject any of `g+r,g+w,o+r,o+w` (`stat.S_IRGRP | S_IWGRP | S_IROTH | S_IWOTH`), not just world-readable. This matches OpenSSH's `IdentityFile` convention. If a future story wants group-readable secret files (k8s Secret volumes mount mode 0o644), relax to `(S_IWGRP|S_IROTH|S_IWOTH)` â€” but keep the policy in one place inside `_resolve_file`.
  - Naming collision avoided: `padrino.llm.secrets` does NOT trip the existing `test_litellm_adapter_does_not_import_db_or_random` firewall, because that test does `mod.startswith("secrets")` and our dotted name starts with `padrino`. Stdlib `secrets` imports would still be caught. If a new module under `src/padrino/` ever happens to start with `secrets.` (e.g. moved to top-level), revisit the firewall.
