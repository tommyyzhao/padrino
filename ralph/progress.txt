## Codebase Patterns

- `create_app(public_surface_only=True)` (or `PADRINO_PUBLIC_SURFACE_ONLY=true`) builds the internet-facing process: only the public router + health router are mounted; private routers and `/metrics` are NOT registered, so private prefixes 404 (route absent), never 401/403. The flag resolves like the other `create_app` knobs: explicit kwarg wins, else `settings.padrino_public_surface_only`. Note `scheduled_gauntlets_router` carries BOTH private CRUD and one public route (`/public/scheduled-gauntlets`); gating the whole router (per AC) drops that public route too â€” the must-work public surface is the enumerated set (`/public/live`, `/public/recent`, `/public/ladder`, `/public/games/{id}/live`).
- `src/padrino/public/` is the new module for Wave 7 public surface. Start each new public module here.
- The public spectator website (`web/dashboard`) must call ONLY `/public/*` endpoints from any nav- or URL-reachable page (it ships against the public-surface-only API). To get a league_id without the private `/gauntlets`, chain public reads: `/public/recent` -> `GET /public/games/{id}.league_id`. The e2e smoke spawn sets `PADRINO_PUBLIC_LEADERBOARD_ANONYMOUS=true` (`tests/e2e/global-setup.ts`) so keyless `/public/*` reads work; the smoke harness does NOT seed broadcastable RECENT games, so `/public/recent` + `/public/live` are empty there (only `/public/ladder` has data from the gauntlet's ratings).
