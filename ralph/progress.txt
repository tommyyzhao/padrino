## Codebase Patterns
(Reusable learnings discovered across wave-3 iterations. Keep this section short
and general — story-specific notes live in the iteration entries below. Patterns
inherited from waves 1-2 are NOT duplicated here; read the archive for those.)

- Markdown-fence quirk on Cerebras GLM-4.7: chat-tuned models routinely wrap their JSON output in ```json ... ``` fences. `padrino.llm.litellm_adapter._strip_code_fence` normalizes this in the impure adapter layer before `parse_agent_response` runs. If a new provider exhibits a different wrapper (e.g. `<json>...</json>` tags, leading "Here is my response:" preamble), extend `_strip_code_fence` AND add a unit test in `tests/llm/test_litellm_adapter.py::test_markdown_code_fence_is_stripped_before_parse`. Do NOT loosen `parse_agent_response` in pure-core — its contract is "give me clean JSON or be rejected".
- Cassette re-recording (`PADRINO_RECORD_LLM=1`): vcrpy's `record_mode="once"` only writes a cassette if the file is MISSING — `rm` the existing yaml first or the run silently no-ops. Recording one provider's `canonical_response.yaml` will typically end up with 3 interactions per cassette because litellm's `_extract_text` raises on gzipped-bytes during the live run (the body decodes fine at REPLAY time), triggering 2 retries. Replay only consumes the first matching interaction, so the extras are dead weight but harmless. Adapter timeout defaults to 5 s; the contract-test fixture extends it to 60 s while `PADRINO_RECORD_LLM=1` so DeepInfra's cold start (15+ s on DeepSeek-V4-Flash) can land. Synthetic-canonical providers (no API key in `.env`) get `ProviderCase(synthetic_canonical=True)` plus a `TODO(US-072)` comment; the parametrize entry stays in the live_llm marker set but skips at runtime.
- The `>=70%` parse-rate quality gate in `tests/integration/test_real_providers.py` exists because the original test passed despite 100% of responses failing to parse (the runner silently coerced them to NOOP / ABSTAIN). Any new real-LLM integration test MUST carry forward this gate or an equivalent — `raw_response is non-empty` is NOT a sufficient assertion.
- `padrino.core.observation_privacy.assert_ranked_observation_safe` enforces the privacy firewall at runtime per game. Wave 3 adds `audit_observation_log_for_seat` (US-078) as the offline auditor that returns findings instead of raising — runtime check and audit check share their forbidden-field list, do NOT drift the two lists out of sync.
- Migration sequencing tests live in TWO files: `tests/db/test_migrations.py` (SQLite-only v1 subset) and `tests/db/test_migrations_all_dialects.py` (SQLite + Postgres via testcontainers). The all-dialects file's `EXPECTED_TABLES` set drives the Postgres fixture's between-test DROP-cascade cleanup; whenever a new table lands in a migration, add it to `EXPECTED_TABLES` in `test_migrations_all_dialects.py` or the full-cycle Postgres test will fail "relation already exists" on the second upgrade head. The SQLite-only file's `EXPECTED_TABLES` is checked as a SUBSET only, so adding tables there is optional.
- Streaming size caps in FastAPI must be enforced at the stream layer, not only after `await request.body()` completes. `Request.stream()` yields chunks as they arrive; accumulating into a list and bailing once the total exceeds the cap is the defensive read path. Honor `Content-Length` up front when present (short-circuit before reading any bytes) AND keep the streamed bound for the chunked-transfer / lying-Content-Length case. Without both, a malicious submitter can force the server to buffer the entire payload before the post-buffer length check fires.
- httpx + ASGITransport supports two body shapes for testing oversized payloads: a `bytes` body with an explicit oversized `Content-Length` header tests the up-front short-circuit, while an async iterator (`AsyncIterable[bytes]`) tests the streamed bound because httpx switches to chunked transfer (no Content-Length) when the body is iterable. Both should converge on the same 413 response.
- Secret-history audits are most reliable when patterns are anchored on specific provider prefixes (`csk-`, `lw`, `sk-`, `sk-ant-`, `ghp_`, `gho_`) AND require a minimum body length, plus a directory-level allowlist for known-synthetic locations: `docs/**` (placeholders like `sk-...`), `tests/**` (cassette-scrub probes per US-072), `pnpm-lock.yaml` / `uv.lock` / `*.lock` (base64 integrity hashes that incidentally contain provider-prefix substrings). The `_API_KEY=` populated-value pattern must restrict the value charset to `[A-Za-z0-9_-]` (no dots) so `.env.example`-shaped placeholders with `...` never match. Print only `<sha>\t<file-path>` to stdout — full matched lines go to a gitignored `.padrino_audit/audit.log` so the audit can never re-leak a secret into a CI log.
- Confidence intervals on binomial proportions at small N must use Wilson, not Wald. `padrino.core.statistics.wilson_score_interval(k, n)` is the closed-form pure-core helper — no scipy. Wald collapses to (0, 0) / (1, 1) at the boundaries and to (negative, >1) for moderate N, both unusable for a public leaderboard claim. Wilson stays inside [0, 1] and non-degenerate at every (k, n). Stick to deterministic bootstrap via `bootstrap_mean_ci(samples, SeededRng(...))` instead of `numpy.random` — the seeded RNG advances a sha256 counter so draws replay bit-for-bit across machines / Python versions, which matters for any test that pins a specific CI band.
- Public/admin route pairs for the same resource (e.g. `/gauntlets/{id}/report` admin + `/public/gauntlets/{id}/report` redacted): keep the redaction shape in a single helper in the impure layer (e.g. `padrino.gauntlets.evaluation.redact_for_public(report) -> dict`) and import that helper from the public route. Do NOT inline `model_dump(exclude=...)` per route — when a new model-identity field gets added, you only update one allowlist. The forbidden-key set should include forward-compatible scrub keys (`model_provider`, `model_name`, `model_version`, `display_name`, `provider`) even if the current schema doesn't carry them, so future schema additions are scrubbed without touching every public route.
- Playwright specs against the smoke-harness backend: when the spec needs specific backend state (a gauntlet ID with rating events, a particular game outcome), prefer `page.route('**/api/path', fulfill({...}))` to intercept with a fixture JSON over depending on whatever the smoke flow happened to produce. The smoke flow's clone count, ranked flag, and adapter (mock vs real) all shift which downstream rows actually populate, and a spec that asserts a specific shape against unpinned state is a flake magnet. Reserve real-backend-driven specs for the cases where the integration itself (CORS, auth fallback, anonymous-flag) is what's under test.

## 2026-05-20 — Wave 3 kickoff (post-Wave-2 audit)

Wave 2 self-reported 22/22 stories passing, but a post-completion audit found:
1. The `test_real_providers_one_game_day` integration test had never been run.

- Cost cap bumped $0.10 -> $0.20 because well-formed responses are longer than truncated failures.

- **Learnings for future iterations:**
  - The bundle's hash chain excludes `event_hash`, `prev_event_hash`, and `created_at` from the hashed body — but `terminal_result` lives at the TOP level of the bundle, NOT inside any event, so it is NOT covered by the hash chain. An attacker can edit `bundle.terminal_result` independently of the event log without invalidating any event hash. The new `inconsistent_terminal` check exists exactly because the hash chain alone does not protect bundle-level fields; any future bundle-level claim (e.g. `bundle.signer_label`) needs its own consistency check or it must be moved into a hashed event.
  - `EXPORT_FORBIDDEN_PAYLOAD_KEYS` and `PUBLIC_TRANSCRIPT_FORBIDDEN_KEYS` are intentionally DIFFERENT sets. Export drops model/provider/rating provenance because the runner should never emit those into the event log.   - The auth pipeline runs rate-limiter BEFORE the route body executes, so even a 413 (raised inside the route) still consumes one rate-limit token. This is what makes the sybil-burst test cheap: posting empty bodies with an oversized Content-Length header is enough to drive the rate limiter to exhaustion without going through the full bundle parse path. If a future story moves the size check upstream of auth (e.g. into a middleware), the sybil test would need to switch to a different request that actually authenticates.
  - To bound a streamed request body in Starlette/FastAPI: iterate `request.stream()` and break (raise HTTPException) once the running total exceeds the cap. Do NOT rely on Starlette's built-in body size limit (configurable on Uvicorn, but not on TestClient/ASGITransport — and the default is generous). The route-level bound is the only portable defense.

- **Learnings for future iterations:**
  - The Wilson score interval is what kept the post-Wave-2 audit's "no degenerate CIs at low N" gate honest. The naive Wald (normal-approximation) interval `p_hat ± 1.96*sqrt(p_hat*(1-p_hat)/n)` collapses to `(1, 1)` when `k = n` and to `(0, 0)` when `k = 0`, even though those outcomes are obviously not certain. Wilson stays non-degenerate at every (k, n) and stays inside [0, 1] without clamping. Any future binomial-CI work in core should reuse `wilson_score_interval` — do NOT add a Wald path.
  - When publishing a deterministic-bootstrap helper for ratings, the cleanest contract is `bootstrap_mean_ci(samples, rng, *, iterations, confidence) -> ConfidenceInterval`. `SeededRng.randbelow(n)` is the deterministic primitive — calling `rng.randbelow(n)` in a tight loop produces the same draws across machines / Python versions because the seeded RNG advances a sha256 counter, not Python's internal PRNG. The single-sample case returns `(p, p, p)` (zero width) rather than failing — caller intent is "report the value, no uncertainty quantifiable" — and the empty-sample case returns all-zero.
  - The `GauntletReport` schema deliberately publishes per-build performance keyed only by `agent_build_id` — never by `model_provider` / `model_name` / `display_name`. This matches US-067's privacy posture for the public model leaderboard. `redact_for_public` exists as forward-compatible defense in depth: today the schema has no leakable columns to strip, but a future field addition (e.g. an admin-only `display_name` summary in `RatingDelta`) gets scrubbed automatically by the existing `_PUBLIC_RATING_DELTA_FORBIDDEN_KEYS` set without having to update every public route. Keep new "model-identity-shaped" keys in that set if they're ever added to a report-adjacent type.
  - Real-action counting for `average_actions_per_seat` must filter `VoteSubmitted` by `payload.is_abstain == True` (abstains DO emit an event, unlike NOOP which emits none). The other three event types — `MafiaKillVoteSubmitted`, `ProtectSubmitted`, `InvestigateSubmitted` — never have an "abstain" variant in v1, so a simple count works. If a future story adds an abstain shape to one of those event types, update the `_REAL_ACTION_EVENT_TYPES` filter logic to mirror the VoteSubmitted treatment.
  - Playwright specs that depend on backend state (gauntlet IDs, rating events) are fragile: the smoke harness produces a single mock-adapter gauntlet whose rating-event population is path-dependent on the harness's clone_count + ranked flag. For dashboard-render specs prefer `page.route` to intercept the API endpoint with a fixture JSON — the spec asserts the page renders correctly given a known shape, decoupled from the smoke harness's gauntlet ID. This pattern also makes the spec runnable against `PADRINO_E2E_SKIP_BACKEND=1` without any extra wiring.
  - `tests/api/test_gauntlet_routes.py` (and similar API route tests) call `POST /model-providers` with `auth_secret_ref="env:CEREBRAS_API_KEY"`, which fails when the env var is unset in an isolated test run. Either run the full suite or set `CEREBRAS_API_KEY=fake DEEPINFRA_API_KEY=fake uv run pytest tests/api/test_gauntlet_routes.py` for ad-hoc development.
