## Codebase Patterns
(Reusable learnings discovered across wave-3 iterations. Keep this section short
and general — story-specific notes live in the iteration entries below. Patterns
inherited from waves 1-2 are NOT duplicated here; read the archive for those.)

- Markdown-fence quirk on Cerebras GLM-4.7: chat-tuned models routinely wrap their JSON output in ```json ... ``` fences. `padrino.llm.litellm_adapter._strip_code_fence` normalizes this in the impure adapter layer before `parse_agent_response` runs. If a new provider exhibits a different wrapper (e.g. `<json>...</json>` tags, leading "Here is my response:" preamble), extend `_strip_code_fence` AND add a unit test in `tests/llm/test_litellm_adapter.py::test_markdown_code_fence_is_stripped_before_parse`. Do NOT loosen `parse_agent_response` in pure-core — its contract is "give me clean JSON or be rejected".
- Cassette re-recording (`PADRINO_RECORD_LLM=1`): vcrpy's `record_mode="once"` only writes a cassette if the file is MISSING — `rm` the existing yaml first or the run silently no-ops. Recording one provider's `canonical_response.yaml` will typically end up with 3 interactions per cassette because litellm's `_extract_text` raises on gzipped-bytes during the live run (the body decodes fine at REPLAY time), triggering 2 retries. Replay only consumes the first matching interaction, so the extras are dead weight but harmless. Adapter timeout defaults to 5 s; the contract-test fixture extends it to 60 s while `PADRINO_RECORD_LLM=1` so DeepInfra's cold start (15+ s on DeepSeek-V4-Flash) can land. Synthetic-canonical providers (no API key in `.env`) get `ProviderCase(synthetic_canonical=True)` plus a `TODO(US-072)` comment; the parametrize entry stays in the live_llm marker set but skips at runtime.
- The `>=70%` parse-rate quality gate in `tests/integration/test_real_providers.py` exists because the original test passed despite 100% of responses failing to parse (the runner silently coerced them to NOOP / ABSTAIN). Any new real-LLM integration test MUST carry forward this gate or an equivalent — `raw_response is non-empty` is NOT a sufficient assertion.
- `padrino.core.observation_privacy.assert_ranked_observation_safe` enforces the privacy firewall at runtime per game. Wave 3 adds `audit_observation_log_for_seat` (US-078) as the offline auditor that returns findings instead of raising — runtime check and audit check share their forbidden-field list, do NOT drift the two lists out of sync.

## 2026-05-20 — Wave 3 kickoff (post-Wave-2 audit)

Wave 2 self-reported 22/22 stories passing, but a post-completion audit found:
1. The `test_real_providers_one_game_day` integration test had never been run.
2. When run, 100% of Cerebras `zai-glm-4.7` responses returned `status=invalid_json` because the adapter did not strip the markdown ```json ... ``` fence the model consistently emits.
3. The runner silently coerced every failing response to a safe-fallback (NOOP / ABSTAIN), so the game appeared to run end-to-end while no real model behavior was being measured.
4. The integration test's assertion (`raw_response or status in _FAILURE_STATUSES`) did not catch this because `invalid_json` was not in the failure statuses set and `raw_response` was always non-empty.

Fix landed pre-Wave-3 on `ralph/padrino-v2` as commit `4e4ed22` and merged forward to `main` and `ralph/padrino-v3`:
- `padrino.llm.litellm_adapter._strip_code_fence` normalizes ```...``` wrappers before parsing.
- 5 new unit tests in `tests/llm/test_litellm_adapter.py` (4 fence variants + unfenced-passthrough).
- A new `>=70%` parse-rate quality gate in `tests/integration/test_real_providers.py` so this class of silent-coercion regression cannot recur.
- Cost cap bumped $0.10 -> $0.20 because well-formed responses are longer than truncated failures.

After the fix: 25/25 calls return `status=ok` against the real Cerebras + DeepInfra providers. All four backend gates remain green; coverage on core/ is 99%.

Wave 3 scope (8 stories, US-071..US-078):

- Cost cap $2.00 with the actual cost printed on success via `capsys.disabled()` so the cap can be tuned without re-running. Test does not mark `pytest.mark.flaky` (the dep isn't installed); the docstring notes real-LLM variance and asks the operator to re-run once on the rare provider-induced DRAW.
- Files changed: `tests/integration/test_real_providers_full_game.py` (new), `ralph/prd.json` (US-071 passes -> true), `ralph/progress.txt` (this entry).
- **Learnings for future iterations:**
  - The runner's `GameOutcome` exposes `final_state`, `event_log`, `llm_calls` — there is NO `outcome.terminal_result` shortcut despite what the US-071 acceptance text says. Reach through `outcome.final_state.terminal_result` (a `str | None`) and `outcome.final_state.terminal_reason`. Mirror this access pattern in US-073 / US-077 / US-078.
  - The submission-event shape in `runner.game_runner._submission_events_for` is the canonical source for "what counts as an action": NOOP emits NO event at all, ABSTAIN emits `VoteSubmitted` with `payload.is_abstain=True`, and the four real action event types are `VoteSubmitted` (is_abstain=False), `MafiaKillVoteSubmitted`, `ProtectSubmitted`, `InvestigateSubmitted`. Re-use this set in US-078's privacy audit and the US-077 evaluation report.
  - `LiteLlmAdapter`'s constructor takes only ONE `auth_secret_ref` (Cerebras) — litellm picks up DeepInfra's key directly from `DEEPINFRA_API_KEY` in the env. If a future story needs the fallback to use a non-env secret (e.g. `file:/path/to/key`), the adapter constructor will need a second secret ref.
  - `pytest.CaptureFixture` is the typed alias for the capsys fixture. `with capsys.disabled(): print(...)` is the idiomatic way to surface a status line from an integration test without breaking pytest's capture for the rest of the run — useful for cost reporting and future US-073 gauntlet summaries.

- **Learnings for future iterations:**
  - vcrpy `record_mode="once"` only writes if the cassette file is MISSING. To re-record an existing cassette, `rm` it first, otherwise the run replays silently and you wonder why nothing changed. CI never records because the fixture flips to `record_mode="none"` when `PADRINO_RECORD_LLM` is unset.
  - Real recordings end up with N interactions per cassette where N is the retry-policy attempt count (default 3). Cause: litellm's `_extract_text` raises on gzipped-bytes during the live run (the body decodes fine at REPLAY time because aiohttp handles Content-Encoding), each retry succeeds at the wire layer, all attempts get written. Replay consumes the first matching interaction and the extras are dead weight but harmless — do NOT trim them by hand; they serve as retry-resilience for any future replay path that re-issues the call.
  - The adapter's `timeout_s` is HARDCODED inside `_build_adapter` in the contract test. For recording, the fixture now branches on `PADRINO_RECORD_LLM` and lifts it to 60 s — DeepInfra's DeepSeek-V4-Flash regularly takes 15+ s on first contact, which would otherwise time out and fail recording. If a third provider with even longer cold start arrives (some self-hosted models warm in 30+ s), bump the recording timeout further; the replay timeout stays at 5 s because cassettes are instant.
  - Malformed-response cassettes are necessarily synthetic — real providers do not emit malformed JSON on demand. The test parametrize set keeps `malformed_response_coerces` running for all 5 providers; only `canonical_response_parses` skips synthetic providers. The story's `pytest.mark.skip` instruction is implemented inside the test body (`pytest.skip(...)`) rather than via `pytest.param(marks=...)` so a future re-record can flip a single flag (`synthetic_canonical=False`) without touching the parametrize list.

## 2026-05-20T23:15 - US-073
- Added `tests/integration/test_real_providers_minigauntlet.py::test_real_providers_minigauntlet`: stands up a file-backed SQLite DB in `tmp_path`, seeds the minimal admin rows via `padrino.demo_gauntlet._seed_minimal_admin`, schedules a 3-clone gauntlet via `create_gauntlet`, runs each child game through `run_game(..., ranked=True, persistence=...)` against the real Cerebras + DeepInfra adapters (one fresh `LiteLlmAdapter` per game, same pattern as `run_demo_gauntlet`), then calls `finalize_gauntlet_if_done` and computes the leaderboard. Gated by `@pytest.mark.integration` and `CEREBRAS_API_KEY` so default CI skips it.
- Assertions: (a) all 3 games reach a terminal state (TOWN / MAFIA / DRAW), (b) `Gauntlet.status` flips to `COMPLETED` and `completed_at` is set, (c) Rating rows exist for `(GLOBAL,'global')`, `(FACTION,'TOWN')`, and `(FACTION,'MAFIA')` scopes and every row has games>0 plus mu/sigma drifted off the OpenSkill defaults (25.0 / 25/3), (d) `compute_leaderboard(...)` returns at least one entry with games>=3 and `provisional=True` per US-045 thresholds (3 games << 30/15/5), (e) aggregate parse-rate across all gauntlet calls >= 70% (carries the post-Wave-2 quality gate forward), (f) total cost <= $6.00 with the actual spend printed via `capsys.disabled()` so the cap can be tuned.
- Files changed: `tests/integration/test_real_providers_minigauntlet.py` (new), `ralph/prd.json` (US-073 passes -> true), `ralph/progress.txt` (this entry).
- **Learnings for future iterations:**
  - `run_demo_gauntlet` runs games but does NOT call `finalize_gauntlet_if_done` — the demo CLI relies on the leaderboard route to do its own aggregation. Any test that wants `Gauntlet.status='COMPLETED'` (US-077 evaluation report will need this) must invoke `finalize_gauntlet_if_done(session, gauntlet_id)` itself after every child game is terminal. The function commits inside its own `session.begin()` block, so callers should NOT wrap it in another transaction.
  - The runner attaches ratings via `update_ratings_for_game` inside the same transaction as `GameTerminated` ONLY when `ranked=True` AND `persistence.league_id` is set AND `persistence.agent_builds` covers every seat. A mini-gauntlet test that omits any of these silently skips the rating updates — making the rating assertions trivially pass against initial-default rows. Always assert that mu/sigma drifted off `INITIAL_MU` / `INITIAL_SIGMA` to catch this regression.
  - `padrino.demo_gauntlet._seed_minimal_admin` is a private helper (underscored) but is the canonical league + provider + model_config + prompt_versions + agent_build seed used by both the demo CLI and this integration test. Importing it from tests is intentional — replicating its setup inline would drift away from the demo path. If US-077 needs a richer seed (multiple agent_builds for cross-build evaluation), refactor `_seed_minimal_admin` into a `padrino.testing` helper rather than copying it.
  - For US-045 provisional thresholds (total_games>=30, town_games>=15, mafia_games>=5), a 3-clone gauntlet ALWAYS produces provisional leaderboard entries — the assertion is `entry.provisional is True`. The reverse (a future stress test that pushes past 30 games) should assert `provisional is False`; same threshold constants live in `padrino.gauntlets.completion`.
  - `tmp_path` is a `pathlib.Path` (function-scoped), NOT a `pytest.TempPathFactory` (session-scoped factory). For per-test SQLite isolation use `tmp_path: Path` and `db_path = tmp_path / "minigauntlet.sqlite"`. Mixing the two annotations works at runtime because pytest provides the right object regardless, but mypy will accept a wrong annotation as long as `# type: ignore[attr-defined]` covers up calls like `.joinpath` — preferring the correct annotation keeps future readers honest.
