## Codebase Patterns
(Reusable learnings discovered across wave-2 iterations. Keep this section short
and general — story-specific notes live in the iteration entries below. Patterns
inherited from wave 1 are NOT duplicated here; read the archive for those.)

- Terminal-game detection (US-049): a game is "terminal" iff `Game.status == 'COMPLETED'`. The runner sets this atomically with the `GameTerminated` event row inside `GamePersistence._persist_terminated_event`. Do NOT add new `event_type='GameTerminated'` queries — filter on `Game.status` instead. The legacy event-scan pattern is gone from `padrino.api` and `padrino.gauntlets` and should not be reintroduced.
- `Game.terminal_result` is a JSON column shaped `{winner, reason, day_terminated}`. The separate `Game.terminal_reason` column was dropped — reason lives inside the JSON. Read it via `game.terminal_result["winner"]`.
- Game-status string values are UPPERCASE to match the existing Gauntlet convention: `CREATED` → `RUNNING` → `COMPLETED`. Story copy showed lowercase as an illustrative example; the codebase uses uppercase.
- Atomic terminal-write seam: `_persist_terminated_event(persistence, stored, state, ranked, day_terminated)` writes the event row, `Game.status='COMPLETED'`, `Game.terminal_result`, and (when applicable) every rating update inside ONE `session.begin()`. Tests assert rollback by monkeypatching `update_ratings_for_game` to raise — the game row and event row must NOT be persisted.
- Seat-backfill seam: `_persist_roles_assigned(persistence, stored)` writes the `RolesAssigned` event row and every `game_seats` row in ONE txn. Requires `GamePersistence.agent_builds` to cover EVERY seat in the payload; partial mappings silently skip seat insertion (the event row still goes through) so callers driving the runner without a per-seat build mapping don't crash. Demo gauntlets and full integration tests always supply the complete map.
- The runner forbidden-imports firewall test (`test_game_runner_does_not_import_forbidden_modules` in `tests/runner/test_game_runner.py`) blocks `random`, `secrets`, `datetime`, `time` imports from `src/padrino/runner/game_runner.py`. If a story needs wall-clock at the runner level, factor the helper into a sibling module (e.g. `padrino.runner.persistence` or similar) and import the helper — don't import the stdlib module from `game_runner.py`.

## 2026-05-14T23:30 - US-049
- Game.terminal_result column changed from String to JSON (`{winner, reason, day_terminated}`); Game.terminal_reason column dropped via Alembic migration 0004. `Game.status='COMPLETED'` is now the canonical terminal filter.
- `GamePersistence` writes one `game_seats` row per `RolesAssigned.payload.assignments[*]` inside the same `session.begin()` as the event row, and flips `Game.status='COMPLETED'` + writes `Game.terminal_result` JSON inside the same txn as the `GameTerminated` event + rating updates. `_backfill_seats` removed from `padrino.demo_gauntlet`; `event_type='GameTerminated'` lookups in `padrino.api.routes.{games,gauntlets}` and `padrino.gauntlets.completion` replaced with `Game.status='COMPLETED'`.
- New tests: `tests/runner/test_game_persistence_seats.py` (seats match RolesAssigned payload; partial agent_builds mapping skips seat insertion), `tests/runner/test_game_persistence_terminal.py` (status + terminal_result committed together; monkeypatched rating failure rolls back event + game row + ratings).
- Files changed: `src/padrino/db/models.py`, `src/padrino/db/repositories/games.py`, `src/padrino/db/migrations/versions/0004_game_terminal_status.py`, `src/padrino/runner/game_runner.py`, `src/padrino/demo_gauntlet.py`, `src/padrino/api/routes/games.py`, `src/padrino/api/routes/gauntlets.py`, `src/padrino/gauntlets/completion.py`, `tests/runner/test_game_persistence_seats.py`, `tests/runner/test_game_persistence_terminal.py`, `tests/api/test_game_routes.py`, `tests/db/test_repositories.py`, `tests/gauntlets/test_completion.py`, `ralph/prd.json`, `ralph/progress.txt`.
- **Learnings for future iterations:**
  - `op.batch_alter_table("games")` with `batch.alter_column("terminal_result", type_=sa.JSON(), postgresql_using="terminal_result::json")` is the right shape to change a SQLite TEXT column to JSON while staying portable for Postgres (US-057). The `postgresql_using` clause is a no-op on SQLite but lets Postgres re-cast existing string rows.
  - `Game.terminal_result` JSON shape is `{winner, reason, day_terminated}` exactly. `day_terminated` comes from `current_phase.day` at the moment GameTerminated is emitted — for MAX_DAYS_REACHED draws that's `ruleset.MAX_DAYS` because the FSM walks through DAY_n_VOTE → NIGHT_n_ACTIONS → TERMINAL keeping day=n.
  - Tests under `tests/gauntlets/test_completion.py` and any new fixtures that hand-craft a terminal game by inserting a `GameTerminated` event row MUST also call `games_repo.update_status(session, gid, status='COMPLETED', terminal_result=...)` — the gauntlet completion helpers now filter on `Game.status`, not on the event row.
